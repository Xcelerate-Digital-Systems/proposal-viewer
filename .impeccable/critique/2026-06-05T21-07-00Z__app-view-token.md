# Harden: app/view/[token]

**Date**: 2026-06-05
**Command**: `harden app/view/[token]`
**Surface**: Public proposal + quote viewer (mobile + desktop)

## Findings & Fixes

### P0 (Critical)

1. **ProposalDecisionPanel: stuck submitting state** — Validation guards (`if (!agree) return`, `if (!signatureData) return`, `if (!reason.trim()) return`) were inside the `try` block AFTER `setSubmitting(true)`. If the user clicked submit and hit one of these guards, the button stayed permanently disabled with no error message. **Fixed**: moved all validation before `setSubmitting(true)`.

2. **CommentsPanel: zero error handling** — `handleSubmit`, `handleReply`, `handleResolve` had no try/catch. If any async callback threw, `setSubmitting(false)` was never reached, permanently locking the UI. No user-facing feedback on failure. **Fixed**: wrapped all three handlers (plus unresolve) in try/catch with `finally` cleanup and inline error alert.

### P1 (High)

3. **SignatureCapture: hardcoded colors break on dark themes** — Toggle buttons used `bg-white`, `text-gray-900`, `text-dim`; canvas stroke was hardcoded `#1f2937`; Clear button and placeholder text used hardcoded grays. All break when the quote uses a dark body background. **Fixed**: all colors now derive from the themed `inputColor`, `inputBg`, and `hairlineColor` props.

4. **FloatingToolbar: silent download failures** — Both composite and direct PDF downloads swallowed errors with `console.error`. Users got no feedback. Raw PDF download also didn't check `response.ok`. **Fixed**: added `downloadError` state that shows a red X icon with "Download failed" title for 3 seconds. Added `response.ok` check.

### P2 (Medium)

5. **SignatureCapture: canvas breaks on device rotation** — Canvas width/height were set once on mount. Rotating a phone or resizing the browser left the drawing area misaligned. **Fixed**: added `resize` event listener that reinitializes the canvas dimensions while preserving existing strokes.

6. **CommentsPanel: text overflow** — Author names and comment content had no overflow protection. Very long names pushed the page link offscreen; long unbroken strings could break the panel layout. **Fixed**: added `truncate` on author names, `break-words` on content, `min-w-0` on the name container.

7. **Missing ARIA labels** — Mobile header buttons (menu, prev page, next page), comments close button, and signature canvas had no accessible labels. **Fixed**: added `aria-label` to all five elements plus `role="img"` on the canvas.

8. **CommentsPanel: reply during submission** — Enter key in the reply input didn't check `replySubmitting` state, allowing duplicate submissions via keyboard. **Fixed**: added `replySubmitting` guard.

## Files Changed

- `app/view/[token]/page.tsx` — aria-labels on mobile nav buttons
- `components/viewer/ProposalDecisionPanel.tsx` — validation before setSubmitting
- `components/viewer/CommentsPanel.tsx` — error handling, ARIA, overflow
- `components/viewer/FloatingToolbar.tsx` — download error feedback
- `components/viewer/SignatureCapture.tsx` — themed colors, resize, ARIA
