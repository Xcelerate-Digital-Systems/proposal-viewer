## Anti-Patterns Verdict

Does this look AI-generated? No. Clean on all absolute bans. No side-stripe borders, no gradient text, no glassmorphism, no hero-metric template, no numbered section markers. Card grid has meaningful variation. Deterministic scan: 0 findings.

## Overall Impression

The documents surface is clean, restrained, and structurally sound. It inherits the shared EntityCard/EmptyState vocabulary well. The biggest opportunity is in the detail layer: upload modal has a raw submit button and no escape-key binding, error recovery for failed page splits is vague, and there's zero help or contextual guidance in the detail view.

## What's Working

1. **EntityCard system**: Shared card component handles five cover states elegantly. Shadow vocabulary is correctly earned.
2. **Upload flow with progress feedback**: Multi-step upload with XHR progress bar and status text is clear.
3. **Semantic color discipline**: Teal exclusively for actions and interactive elements. Neutrals follow ink/prose/dim/faint hierarchy.

## Priority Issues

### [P1] Upload modal breaks Button primitive and has no Escape key
DocumentUploadModal.tsx:186-192 uses raw button with bg-teal hover:bg-[#01434A]. Close button (line 123) is also raw. No Escape key binding.
Fix: Replace with Button primitive. Add useEffect for Escape key.

### [P1] Error recovery for page-split failure is vague and lossy
When PDF upload succeeds but page splitting fails (DocumentUploadModal.tsx:93-99), user sees "try re-uploading" with no retry. Uploaded PDF remains orphaned in storage.
Fix: Add "Retry split" action. Show persistent banner instead of transient toast.

### [P1] Icon-only buttons use title but lack aria-label
Seven icon-only buttons across the surface use title but no aria-label. WCAG 1.1.1 violation.
Fix: Add aria-label matching existing title text.

### [P2] No keyboard shortcuts or bulk actions for power users
No global shortcuts. No bulk-select. Sorting fixed to updated_at.
Fix: Add Ctrl+K for search. Add sort dropdown. Design bulk-select UX.

### [P2] Contrast failure: text-faint on white backgrounds
text-faint (#8C8C8C) on white has ~3.3:1 contrast, below WCAG AA 4.5:1 threshold.
Fix: Promote to text-dim (#6B7280, ~4.6:1) for body-size text.

## Persona Red Flags

**Alex (Power User)**: No keyboard shortcuts. No bulk-select. Sorting locked to updated_at. Delete requires individual confirmation.

**Sam (Accessibility-Dependent)**: Seven icon-only buttons lack aria-label. Modal has no Escape key. DocumentListRow div is click target but has no role="button", tabIndex, or focus-visible ring. Page count hidden from screen readers on narrow viewports.

**Sarah (Agency Account Manager)**: No favorites or pinning. No folders/tags/client filtering. Recently-edited hardcoded to 3 items. Flat list with text-search-only at scale is a scanning exercise.
