## Priority Issues

### [P1] Testimonials are placeholder social proof

All four quotes attribute to "Founding Agency" with generic role titles. An agency director will immediately recognize these as fabricated. Fake-looking social proof is worse than no social proof.

**Why it matters:** The mock frames and specific copy build real trust. The testimonials section undercuts it.

**Fix:** (a) Use real names/agencies with permission, (b) label as "Beta feedback, agency names withheld," or (c) remove the section until real testimonials exist.

**Suggested command:** `/impeccable clarify` (copy fix) or manual data update

### [P1] Hero SVG path animation budget is high

60 animated SVG `<motion.path>` elements (30 per FloatingPaths instance x 2) with `repeat: Infinity` on page load. Each is an independent framer-motion timeline. On low-powered devices this could cause first-paint jank, exactly when first impressions form.

**Why it matters:** A B2B tool should signal reliability. A janky first paint signals the opposite.

**Fix:** Reduce to 12-15 paths per instance, or use a single pre-rendered SVG with CSS animation instead of 60 framer-motion timelines.

**Suggested command:** `/impeccable optimize components/marketing/AnimatedHero.tsx`

### [P2] Button implementation inconsistency

Three different button patterns on one page: hero uses `buttonClasses()`, final CTA uses raw `<a>` with hand-written classes, pricing uses another inline pattern. Focus ring styling differs between them.

**Why it matters:** Accessibility inconsistency (focus rings look different per button) and maintenance drift.

**Fix:** Use `buttonClasses()` everywhere, or add a `primary-on-dark` variant for dark-section CTAs.

**Suggested command:** `/impeccable polish app/home`

### [P2] No active-state anchor highlighting

Header anchor links (Features, Pricing, FAQ) have no scroll-spy. When scrolled to FAQ, the "FAQ" link doesn't visually indicate the current section.

**Why it matters:** Missed wayfinding on a 10-section page. The infrastructure exists (useScrolled hook) but isn't extended.

**Fix:** Add IntersectionObserver-based active state to anchor links.

**Suggested command:** `/impeccable craft scroll-spy`

### [P3] h-[100dvh] overflow-y-auto root container

The entire page renders in a div with `overflow-y-auto` instead of natural document scroll. This may break native scroll restoration, Find-on-page, iOS status-bar scroll-to-top, and browser anchor behavior.

**Why it matters:** Subtle but real UX risk across browsers. The ContainerScroll component requires this for its scroll tracking.

**Fix:** Test across browsers. Consider moving to native scroll with a scoped scroll container only around the ContainerScroll section.

**Suggested command:** `/impeccable audit app/home` (browser-level testing)

- ContainerScroll laptop frame uses hardcoded `border-[#6C6C6C]` and `bg-[#222222]` instead of tokens.
- The footer blur-up animation and the FAQ ScrollReveal create two competing reveals in proximity.
- If the video fails to load, the user sees an empty rounded rectangle (no fallback content).

## Questions to Consider

1. **Are the placeholder testimonials doing more harm than good?** The mock frames and specific copy already build trust. Would removing testimonials until real ones exist make the page stronger?

2. **Is the per-letter hero animation earning its cost?** 35 independent spring animations + 60 SVG paths. A two-line fade-up would look 90% as good at 2% of the render cost.

3. **What happens when PUBLIC_SIGNUP_ENABLED flips?** The waitlist form becomes a link. The final CTA loses its interactive commitment moment. Has the post-signup version been designed?

4. **What authority signal would make Dana sign up?** No logos, no "trusted by X," no case studies. Is product-feature persuasion enough for a B2B purchasing decision?
