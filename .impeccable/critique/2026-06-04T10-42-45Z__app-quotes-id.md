## Anti-Patterns Verdict

**LLM assessment**: Clean. No AI slop tells. The 4 SectionGroup headers use sentence-case `text-xs font-semibold text-dim` with a functional chevron, not uppercase tracked eyebrows. They serve a real UX purpose (collapse/expand) and are clearly organizational chrome, not decorative scaffolding.

**Deterministic scan**: Zero findings. All quote builder files clean across regex rules.

## Overall Impression

The quote builder has improved meaningfully across all three passes. The 13-card scroll wall is now 4 collapsible groups with a sticky nav, cutting cognitive load. Save behavior is consistent (blur auto-save), the Draft status change is guarded, input validation catches bad values, and all interactive elements use the Button primitive with proper ARIA attributes. The line items table now has full ARIA table semantics for screen readers. The remaining gaps are keyboard shortcuts (Cmd+S, Tab flow) and the absence of undo functionality.

## What's Working

1. **Section grouping with sticky nav** is the biggest UX improvement. 4 groups with tight internal spacing (12px) and generous separation (32px) create visual rhythm. The nav tracks the active group via IntersectionObserver and provides one-click jump-to with auto-expand. Collapse animation uses CSS grid transition with reduced-motion support.

2. **Consistent save behavior** across all quote-specific sections. ProjectDetailsSection and ScopeOfWorksSection now auto-save on blur with inline "Saving…" indicators. PricingSettingsSection toggles auto-save on change. No more mixed auto-save + explicit Save button patterns.

3. **Line items table accessibility** is now solid. ARIA table roles (table, rowgroup, row, columnheader, cell), labeled inputs (Item name, Description, Quantity), contextual remove labels ("Remove {item name}"), sr-only Actions header. Screen readers can navigate this as a proper data table.

## Priority Issues

### [P2] No keyboard shortcuts or accelerators

Still no Cmd+S, no Tab-to-next-row in line items, no Enter-to-add-row. The section nav and collapsible groups help navigation, but the power user still relies on mouse-only interactions for editing.

**Why it matters**: The Alex persona building 10 quotes/day needs keyboard efficiency.

**Fix**: Add Cmd+S global save. Make line items Tab-navigable in row order (skip Total and Delete). Enter on last row auto-adds.

**Suggested command**: `/impeccable harden app/quotes/[id]`

### [P2] No undo functionality despite EditorUndoProvider in layout

EditorUndoProvider wraps the layout but zero sections consume it. Draft status change to "Draft" is now confirmed but still destructive. No way to recover from accidental field edits that auto-save on blur.

**Why it matters**: Auto-save on blur is the right UX, but it means the user has zero recovery window. One wrong keystroke + blur = saved.

**Fix**: Wire EditorUndoProvider into auto-save sections. Store previous value on focus, restore on Cmd+Z within a brief window.

**Suggested command**: `/impeccable harden app/quotes/[id]`

### [P3] Stale subtotal in Tax & Deposit section

PricingSettingsSection fetches line items independently from QuoteLineItemsSection. Editing a line item won't update the Tax & Deposit subtotal until the proposal row itself changes (which it doesn't on pricing page saves).

**Why it matters**: Financial data should never show stale numbers. If the user edits a line item and then scrolls to Tax & Deposit, the subtotal may be wrong.

**Fix**: Pass a refresh signal from QuoteBuilderV2 to PricingSettingsSection when line items save.

**Suggested command**: `/impeccable harden app/quotes/[id]`

### [P3] No contextual help or onboarding tour

No tooltips, no guided tour, no "what's this?" help for the quote builder. The section descriptions help but are static. A first-time user has no guidance beyond placeholder text.

**Why it matters**: The Jordan (first-timer) persona would benefit from a guided tour. react-joyride is already in the project but not wired into quotes.

**Fix**: Add a quote builder tour via react-joyride targeting the section groups, line items, preview, and key actions.

**Suggested command**: `/impeccable onboard app/quotes/[id]`

## Minor Observations

- PricingSettingsSection fetches line items independently; stale subtotals possible after line item edits
- CurrencyInput on unit price vs raw `<input type="number">` on qty is a visual inconsistency
- LineItemsLibraryBar dropdown uses `absolute z-50`; could be clipped by scroll containers (other dropdowns use createPortal)
- Focus ring at `focus:ring-teal/30` (30% opacity) is project-wide; could be strengthened for WCAG AAA but is consistent

## Questions to Consider

1. Should the section nav remember which groups were open/closed across sessions (localStorage)?
2. Would a "quick add" keyboard shortcut (e.g. / or Cmd+K to search sections/actions) help power users more than individual shortcuts?
3. The stale subtotal is a data architecture issue. Would lifting pricing state to a shared context (or making usePricingEditor emit events) be worth the complexity?
